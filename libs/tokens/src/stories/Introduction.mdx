# Theta Design Tokens

Design tokens define visual properties like colors, spacing, and typography as reusable data that can be transformed into code for any platform.

Most developers use pre-built components from `@theta/web` or `@theta/native`. This documentation covers:
- Token system architecture
- Creating custom components
- Building theme variations
- Extending the design system

## Getting Started

If you're new to the token system, start with the [Token Guide](/docs/getting-started-token-guide--docs) which covers:
- The three-tier architecture (primitives → semantic → component)
- Token format and naming conventions
- How to add and modify tokens
- Platform-specific outputs

## Platform Support

Tokens are automatically transformed into platform-specific formats:
- **Web**: CSS custom properties and TypeScript types
- **React Native**: JavaScript objects with platform-specific values
