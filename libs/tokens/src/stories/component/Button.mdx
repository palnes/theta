## Usage Examples

### CSS Variables
```css
.button {
  height: var(--cmp-button-height-medium);
  padding: 0 var(--cmp-button-paddingx-medium);
  font-size: var(--cmp-button-fontsize-medium);
  font-weight: var(--cmp-button-fontweight);
  border-radius: var(--cmp-button-radius-medium);
}

.button--primary {
  background-color: var(--cmp-button-color-primary-background-default);
  color: var(--cmp-button-color-primary-text-default);
}

.button--primary:hover {
  background-color: var(--cmp-button-color-primary-background-hover);
}
```

## Design Guidelines

### Size Variants
- **Small**: Compact buttons for constrained spaces
- **Medium**: Default size for most use cases
- **Large**: Prominent actions or touch-friendly interfaces

### Color Variants
- **Primary**: Main actions and CTAs
- **Secondary**: Supporting actions
- **Ghost**: Minimal visual weight with border
- **Danger**: Destructive or high-impact actions

### States
- **Default**: Normal resting state
- **Hover**: Mouse hover interaction
- **Active**: Pressed/active state
- **Disabled**: Non-interactive state

### Accessibility
- All buttons include focus outline tokens for keyboard navigation
- Color combinations meet WCAG contrast requirements
- Minimum touch target sizes for mobile interfaces
