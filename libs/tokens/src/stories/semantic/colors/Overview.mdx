# Semantic Colors Overview

Our semantic color system provides meaning and context to UI elements. These colors are role-based and adapt to different themes automatically.

## Color Categories

### [Action Colors](/story/token-reference-semantic-colors-action--docs)
Interactive elements like buttons, links, and form controls. These guide user actions and provide feedback.

### [Surface Colors](/story/token-reference-semantic-colors-surface--docs)
Background colors for various UI surfaces and containers. These create visual hierarchy and define different areas.

### [Text Colors](/story/token-reference-semantic-colors-text--docs)
Typography colors for different contexts and importance levels. These ensure readable and accessible text.

### [Border Colors](/story/token-reference-semantic-colors-border--docs)
Colors for edges, dividers, and separators. These create structure and visual boundaries.

### [Status Colors](/story/token-reference-semantic-colors-status--docs)
System states and feedback colors. These communicate the current status of actions, processes, or information.
